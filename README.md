# Distributed Quiz Management System

## Project Description

This project is a distributed web-based quiz platform that supports quiz creation, quiz participation, result tracking, leaderboards, and real-time live quiz rooms. It is built using a separate frontend and backend, with Redis, PostgreSQL, Celery, and Nginx used to provide distributed coordination, background processing, persistence, and request routing.

The system is designed as a client-server application with additional distributed components:

- React frontend for the user interface
- FastAPI backend for APIs and live quiz management
- Redis for caching, pub/sub, rate limiting, and leaderboards
- PostgreSQL for persistent data storage
- Celery for background task execution
- Nginx for reverse proxying and request routing

## Objectives

- Build a distributed client-server quiz application
- Support real-time multiplayer live quiz sessions
- Maintain shared room state across multiple users
- Use background workers for asynchronous processing
- Provide real-time event delivery using WebSockets
- Use Redis for distributed caching, pub/sub, rate limiting, and locking
- Use PostgreSQL for durable storage of quizzes, results, and room events
- Support scalability through separate frontend, backend, worker, cache, and database services

## Procedure to Run the Project

### Method 1: Run Using Docker Compose

This is the easiest and recommended way to run the full system.

1. Clone the repository:

```bash
git clone <repository-url>
cd distributed_project
```

2. Create environment files:

```bash
cp .env.example .env
cp backend/.env.example backend/.env
```

3. Start the complete application:

```bash
bash run-local.sh
```

4. Open the application in your browser:

```text
http://localhost:8080
```

5. To stop the application:

```bash
bash stop.sh
```

### Method 2: Run Services Manually

Use this approach if you want to run each service separately for development or debugging.

1. Clone the repository:

```bash
git clone <repository-url>
cd distributed_project
```

2. Create the backend environment file:

```bash
cp backend/.env.example backend/.env
```

3. Run the setup script:

```bash
bash setup.sh
```

4. Start Redis.

Example on macOS with Homebrew:

```bash
brew install redis
redis-server
```

5. Start PostgreSQL and create the required database named `quizdb`.

6. Start the backend in a new terminal:

```bash
bash start-backend.sh
```

7. Start the Celery worker in another terminal:

```bash
bash start-celery.sh
```

8. Start the frontend in another terminal:

```bash
cd frontend
npm start
```

9. Open the frontend in your browser:

```text
http://localhost:3000
```

## Optional ngrok Setup

If you want to share your local application with others over the internet for testing or demo purposes, you can use ngrok.

1. Make sure the application is already running.
2. Start ngrok:

```bash
bash start-ngrok.sh
```

3. Use the public URL generated by ngrok.

ngrok creates a secure public tunnel to the locally running application, which is useful when remote users need access without deploying the project to a public server.

## Deployment Notes

- Local Docker deployment uses `ssl/docker-compose.yml`
- Production deployment uses `docker-compose.prod.yml`
- Nginx routes frontend traffic to the React app and API/WebSocket traffic to the backend
- Redis and PostgreSQL act as shared infrastructure services across backend and worker processes

## Default Local Ports

- Frontend: 3000
- Backend: 8000
- Redis: 6379
- PostgreSQL: 5432
- Nginx local: 8080
- Nginx HTTPS local: 8443
